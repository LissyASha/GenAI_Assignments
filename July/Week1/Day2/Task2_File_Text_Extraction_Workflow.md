# Task 2: File Text Extraction using Langflow, Groq and Postman

## Objective

Build a document question-answering workflow that can:

1. Read a PDF or supported document.
2. Extract the document text.
3. Ask questions about the document.
4. Generate answers using Groq LLM.
5. Validate the same workflow using Postman.

## Workflow Architecture

Read File -> Prompt Template <- Chat Input -> Groq -> Chat Output

The workflow uses:

* Read File: Supplies document content.
* Chat Input: Supplies the user's question.
* Prompt Template: Combines both inputs.
* Groq: Generates the response.
* Chat Output: Displays the answer.

## Step 1 – Read File Component

Purpose:

* Upload and read PDF, TXT, DOCX, or supported file types.
* Extract document text using Raw Content.

## Step 2 – Chat Input Component

Purpose:

* Accept user questions dynamically.

Examples:

* Summarize this invoice.
* What is the invoice amount?
* Who is the customer?

## Step 3 – Prompt Template Component

Example template:

You are an intelligent document assistant.

Document:
{document}

Question:
{question}

Answer the question only using information present in the document.

Variable Mapping:

* document -> Read File (Raw Content)
* question -> Chat Input

## Step 4 – Groq Component

Configuration:

* Provider: Groq
* Model: llama-3.3-70b-versatile
* API Key: Groq API Key

Responsibilities:

* Read document context.
* Answer questions.
* Generate summaries.
* Extract information.

## Step 5 – Chat Output Component

Displays the final response generated by the model.

## End-to-End Execution Flow

PDF Upload
↓
Read File
↓
Prompt Template
↑
Chat Question
↓
Groq
↓
Chat Output

## Export and Validate using Postman

1. Run the workflow successfully in Langflow.
2. Click API → cURL.
3. Copy the generated cURL command.
4. Open Postman.
5. Select Import → Raw Text.
6. Paste the cURL command.
7. Execute the request.

## Validation

Verify:

* Document text matches Langflow output.
* Summary matches.
* Question answering matches.
* HTTP status is 200 OK.

## Conclusion

This workflow demonstrates:

* Document ingestion
* Text extraction
* Prompt engineering
* LLM integration using Groq
* Dynamic question answering
* API testing using Postman
